### 📝 הפרומפט לשליחה ל-Claude Sonnet

```
אני מצרף קובץ ZIP של תיקיית app של פרויקט Android שלי.
אנא נתח את הקוד וצור עבורי את החלק התיאורטי של ספר פרויקט גמר בפורמט הבא:

הספר צריך להכיל את הפרקים הבאים (בעברית):

1. **מבוא**
   - שם הפרויקט
   - תיאור קצר של הפרויקט
   - קהל היעד
   - הסיבות לבחירת הנושא

2. **מטרת המערכת**
   - הסבר בראיית המשתמש – מה המערכת עושה ולאיזה צורך היא עונה

3. **יכולות המערכת (Feature List)**
   - רשימת כל האפשרויות שהמערכת מעניקה למשתמש

4. **תהליך המחקר**
   - טכנולוגיות שנעשה בהן שימוש (ספריות, APIs, כלים)
   - סקירת מצב קיים בשוק – אפליקציות דומות וחידושים בפרויקט
   - אתגרים מרכזיים שהתמודדת איתם ופתרונות

5. **מדריך למשתמש**
   - תיאור כל מסך: שם המסך, מה הוא מציג, מה ניתן לבצע בו
   - תיאור כל כפתור / אלמנט תצוגה שאינו מובן מאליו
   - תיאור מחזור חיים (onPause, onResume וכו') אם רלוונטי

6. **הרשאות**
   - רשימת הרשאות Android שהאפליקציה מבקשת והסבר לכל אחת

7. **מבנה / ארכיטקטורה**
   - תיאור המחלקות הראשיות ותפקידן
   - קשרים בין המחלקות
   - תיאור כל Activity, Service, Fragment

8. **שמירת נתונים**
   - כיצד הנתונים נשמרים (SQLite / SharedPreferences / קבצים)
   - סכמת Database אם קיים (טבלאות, שדות, קשרים)

9. **אתגרים מרכזיים**
   - הבעיות המרכזיות שנתקלת בהן
   - הפתרונות שיישמת
   - חידושים ייחודיים בפרויקט

פרמטרים נוספים:
- כתוב בעברית תקינה
- כל פרק יהיה לפחות פסקה מלאה אחת
- פרק המבוא יהיה לפחות עמוד מלא
- השתמש בכותרות מסודרות
- אל תכתוב את הרפלקציה האישית והביבליוגרפיה – אלה יושלמו ידנית

**הוראות חשובות לגבי תמונות:**
אינך יכול לצרף תמונות בפלט הטקסטואלי.
בכל מקום שנדרשת תמונה, כתוב placeholder בפורמט הבא בשורה נפרדת:

📸 [הכנס כאן תמונה: <תיאור מדויק של מה שצריך להופיע בתמונה>]

דוגמאות לשימוש:
- בתרשים זרימה בין מסכים: 📸 [הכנס כאן תמונה: תרשים זרימה בין מסכי האפליקציה]
- בתיאור מסך: 📸 [הכנס כאן תמונה: צילום מסך של MainActivity]
- בתרשים מחלקות: 📸 [הכנס כאן תמונה: תרשים מחלקות UML של הפרויקט]
- בסכמת DB: 📸 [הכנס כאן תמונה: סכמת בסיס הנתונים עם הטבלאות והקשרים]

חובה להוסיף placeholder בכל מקום שנדרש ויזואל – הסטודנט יחליף אותם בתמונות האמיתיות.
```

### 🖼️ שלב ד׳ – הוספת תמונות ידנית (חובה!)

ה-AI **אינו יכול לצרף תמונות** לספר – הוא יסמן לך בדיוק **היכן** להכניס כל תמונה.  
בפלט של Claude תופיע הוראה כזו בכל מקום שצריך תמונה:

```
📸 [הכנס כאן תמונה: <תיאור התמונה>]
```

**לאחר ההדבקה ב-Word, חפש את כל ה-`📸` ועשה:**

| סוג תמונה | מה להכניס | איך |
|-----------|-----------|-----|
| **צילום מסך של האפליקציה** | Screenshot מהאמולטור / מכשיר אמיתי | Windows: `Win + Shift + S` / Android Studio: צלם מהאמולטור |
| **תרשים זרימה בין מסכים** | תמונה שציירת / צילום מסך של draw.io | ציור ידני או כלי דיאגרמה |
| **תרשים מחלקות** | תמונת UML | ב-Android Studio: Diagrams → Show Diagram |
| **סכמת Database** | צילום מסך מ-DB Browser / תרשים ידני | DB Browser for SQLite |

**שלבים להכנסת תמונה ב-Word:**
1. לחץ במקום ה-`📸 [הכנס כאן תמונה: ...]`
2. **מחק את שורת ה-placeholder** כולה
3. לחץ **Insert → Pictures → This Device**
4. בחר את התמונה המתאימה
5. גודל מומלץ: **רוחב עמוד** (Wrap Text → In Line with Text)

## חלק 2 – יצירת ספר קוד עם Python

### 📌 מה הסקריפט עושה?

הסקריפט `extract_project.py` סורק את תיקיית ה-`app` שלך ומייצר קובץ Word בשם **`ProjectBook.docx`** המכיל:

- ✅ כל קבצי **`.java`** בפרויקט
- ✅ כל קבצי **`.xml`** שנמצאים ב-`res/layout` בלבד
- ❌ **לא** כולל קבצי XML אחרים (colors.xml, strings.xml וכו')
- ❌ **לא** כולל תיקיות `build`, `.gradle`, `.idea`

לכל קובץ מוצג:
- שם הקובץ + סוג (JAVA / XML LAYOUT)
- הנתיב היחסי שלו בפרויקט
- קוד המקור המלא בגופן `Courier New`
